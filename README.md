# Home-Security-Monitor
1、引言
1.1 项目背景
目前越来越多家庭出现了一些安全事故，如家里线路短路发生火灾，煤气泄漏、非法闯入等场景，基于家庭安装的摄像头形成动态视频，结合AI分析，训练算法，解决家庭AI安全监控问题，是值得探索的主题。
应用视频技术，结合AI分析，训练算法，实现对家庭重点区域和特定场景的监控、识别与告警，1）通过视频AI，对烟雾及火焰检测，发现就标注报警。 2）通过视频AI，在特定时段对进入人员进行检测，发现陌生人员闯入就标注报警。另外就是要注意的是，重复的报警要过滤合并。
随着现代家庭安全意识的提升，家庭火灾隐患及非法入侵等安全事故频发。传统安防设备往往只能事后查看录像，缺乏主动预警能力。本项目结合深度学习（YOLO算法）与计算机视觉技术，旨在打造一套低成本、高效率的AI家庭安全监控系统，实现事前预警和事中告警。
1.2 项目目的
1.实现基于视频流的24小时实时烟雾与火焰检测，保障家庭消防安全。
2.实现在特定时段（如深夜23:00-06:00）的异常人员闯入检测，保障家庭财产与人身安全。
3.实现报警事件的智能合并与去重机制，避免无效告警对用户造成干扰。

3、功能性需求分析
功能点1：实时视频流获取。系统能够驱动本地/局域网摄像头，实时读取视频帧并展示。
功能点2：全天候火灾隐患检测。系统后台运行AI识别算法，一旦识别到火焰（Fire）或烟雾（Smoke），立即触发安全告警。
功能点3：特定时段入侵检测。系统仅在预设时段（如23:00至06:00）开启人员（Person）检测，若发现人员闯入，触发入侵告警。
功能点4：防抖与报警合并机制。在持续发生同类危险事件时，系统能在指定时间窗口内（如10秒）对连续告警进行过滤合并，仅记录一次核心事件。
功能点5：历史报警日志记录。发生告警时，系统需自动抓拍当前异常画面，并将时间、类型、图片路径存入本地数据库。
4、非功能需求分析

4.1 性能需求
视频处理帧率需保持在 15-30 FPS 以上，确保画面流畅；报警延迟时间（从画面出现异常到系统记录）不超过 2 秒。
4.2 可靠性需求
系统能够连续运行24小时不出现崩溃（内存泄漏问题）；在摄像头意外断开时能给出提示而不是直接闪退。
4.3 安全性需求
报警日志保存在本地，不允许非管理员篡改；对于系统配置（如修改监控时间），只有管理员有权操作。
